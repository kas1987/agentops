# Learnings Quality Checklist

Use this checklist before committing a new learning note.

## Source Integrity

- [ ] Source title and URL are included.
- [ ] Author/creator and publication date are captured.
- [ ] Claims are grounded in source text, not assumptions.

## Structure Quality

- [ ] Core thesis is clear in 1-3 sentences.
- [ ] Key claims are listed as distinct bullets.
- [ ] Practical implications are split into individual and team/org levels.
- [ ] Decision rules are included for real workflow use.

## Evidence Quality

- [ ] At least one evidence signal exists per major claim.
- [ ] Confidence level (high/medium/low) is stated.
- [ ] Risks/counterpoints are documented.

## Actionability

- [ ] Includes a 7-day action plan.
- [ ] Includes a 30-day action plan.
- [ ] Includes at least one measurable success metric.

## Writing Quality

- [ ] Uses concise headings and scannable bullets.
- [ ] Uses consistent terminology across the document.
- [ ] Avoids benchmark-only framing when workflow fit is the real decision.

## Repository Hygiene

- [ ] Filename is kebab-case and descriptive.
- [ ] Entry is linked in docs/learnings/README.md.
- [ ] Entry is reachable from docs/INDEX.md.
